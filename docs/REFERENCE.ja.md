## 1. 概要

PyxelBasic は、Pyxel 上で動作する行番号方式の古典的 BASIC インタプリタです。

- 行番号必須。プログラムは行番号順に実行される。
- 1 行 1 命令（マルチステートメント `:` は非対応）。
- 大文字小文字は区別しない（`print` も `PRINT` も同じ）。命令・関数・変数名は内部的に大文字化される。
- プログラムはテキストファイル（`.bas`）として保存・読み込みできる。

---

## 2. 起動と画面

```
python main.py            通常起動（編集モード）
python main.py hello      起動時に samples/hello.bas を読み込む
```

- 画面サイズ: 256 x 256 ピクセル、60 FPS。
- 文字は 1 文字 4 x 6 ピクセル。テキスト画面は **64 桁 x 42 行**。
- 画面はフルスクリーンエディタ。書いた文字は残り、カーソルは出力と編集で共用
  （`PRINT` はカーソル位置から流れ、`LOCATE` はカーソルを移動するだけ）。

### 2.1 スクリーン編集

カーソルキーで自由に移動してその場で編集でき、Enter でカーソル位置の**論理行**を
投入する（論理行は複数の画面行に折り返すことがある）。これにより `LIST` で一覧を
出し、カーソルを目的の行へ動かして一部を書き換え、Enter で修正、という操作ができる。

| キー | 動作 |
|---|---|
| 矢印キー | カーソル移動 |
| 印字可能キー | カーソル位置に文字入力 |
| `Insert` | 挿入／上書きモードの切替（カーソル形状が変化：下線=挿入、枠=上書き） |
| `Backspace` | カーソルの左の文字を削除 |
| `Delete` | カーソル位置の文字を削除 |
| `Home` / `End` | 論理行の先頭 / 末尾へ移動 |
| `Enter` | カーソル位置の論理行を投入し、1 論理行下へ移動 |

## 3. 動作モデル

### 3.1 行の入力・編集

Enter を押すと、カーソル位置の論理行が内容によって次のように扱われる。

| 入力 | 動作 |
|---|---|
| 行番号 + 命令（例 `10 PRINT "HI"`） | その行番号にプログラム行を格納 |
| 同じ行番号で再入力 | その行を上書き |
| 行番号のみ（例 `10`） | その行を削除 |
| 命令・コマンドのみ（行番号なし） | ダイレクトコマンドとして即時実行（出力はカーソル位置から） |

### 3.2 実行モデル

- `RUN` でプログラムを行番号順に実行する。
- 実行は 1 フレームあたり最大 800 命令ずつ進む。`PRINT` などフレーム区切り対象の命令や `VSYNC` に達すると、そのフレームの実行を打ち切り次フレームへ継続する（[フレーム制御](#10-フレーム制御vsync)参照）。
- `INPUT` に達すると入力待ちになり、ユーザーが 1 行入力して Enter を押すと再開する。
- `END` / `STOP`、または最終行まで到達すると実行を終了し、`OK` を表示して編集モードへ戻る。
- 実行中にエラーが発生すると `?ERROR <コード> in line <行番号>: <内容>` を表示して停止する（コードは第13章参照）。

## 4. ダイレクトコマンド

プロンプトで行番号を付けずに入力する管理用コマンド。

| コマンド | 説明 |
|---|---|
| `RUN` | プログラムを最初から実行する |
| `LIST` | プログラム全体を表示する |
| `LIST n` | 行番号 n のみ表示 |
| `LIST n,m` | 行番号 n から m までを表示 |
| `NEW` | プログラムを全消去する（フレーム区切り設定も初期化） |
| `RENUM` | 行番号を 10 から 10 刻みで振り直す |
| `RENUM start` | start から 10 刻みで振り直す |
| `RENUM start,step` | start から step 刻みで振り直す |
| `SAVE "名前"` | `samples/名前.bas` に保存する |
| `LOAD "名前"` | `samples/名前.bas` を読み込む（現在のプログラムは破棄） |

- `RENUM` は `GOTO` / `GOSUB` / `THEN` の直後にある行番号も自動で付け替える。
- `SAVE` / `LOAD` のファイル名に拡張子がなければ `.bas` を補う。保存先は `samples/` ディレクトリ。
- 上記以外の命令（`PRINT`、代入など）もプロンプトで直接実行できる（ダイレクト実行）。

## 5. データ型

| 型 | 説明 | 真偽 |
|---|---|---|
| 数値 | 整数・実数を区別しない（Python の int / float に準拠） | 真 = -1、偽 = 0 |
| 文字列 | 文字の並び。末尾 `$` 付きの変数・関数で扱う | - |

- 比較演算・論理演算の結果は **真 = -1、偽 = 0** を返す。
- 整数値を持つ実数は表示時に整数として表示される（例: `4.0` → `4`）。

## 6. 変数と配列

### 6.1 変数

- **数値変数**: 名前は英字で始まる英数字列（例 `A`、`X1`、`COUNT`）。未定義時の値は `0`。
- **文字列変数**: 名前の末尾に `$` を付ける（例 `A$`、`NAME$`）。未定義時の値は `""`。
- 代入は `LET` を付けても省略してもよい。

```basic
10 LET A = 10
20 B = A * 2
30 N$ = "HELLO"
```

- 数値変数に文字列を代入するとエラー。
- 文字列変数に数値を代入した場合は自動的に文字列へ変換される。

### 6.2 配列

- `DIM` で宣言する。添字は **0 始まり**。`DIM A(10)` は `A(0)`～`A(10)` の 11 要素。
- 多次元配列も宣言できる。`DIM M(3,3)` は `M(0,0)`～`M(3,3)` の 16 要素。
- 数値配列は `0`、文字列配列（末尾 `$`）は `""` で初期化される。
- `DIM` せずに配列を参照・代入すると、各次元 10（0～10）で**暗黙的に確保**される。
- 添字が範囲外の場合はエラー。
- 同名のスカラー変数と配列は別物として扱われる（`A` と `A(0)` は無関係）。

```basic
10 DIM A(5)
20 FOR I = 0 TO 5
30 A(I) = I * I
40 NEXT I
50 PRINT A(3)        ' 9 を表示
```

## 7. 演算子

### 7.1 一覧

| 種類 | 演算子 |
|---|---|
| 算術 | `+` `-` `*` `/` `MOD` `^` |
| 比較 | `=` `<>` `<` `<=` `>` `>=` |
| 論理 | `AND` `OR` `NOT` `XOR` |

### 7.2 優先順位（低い順 → 高い順）

1. `OR` `XOR`
2. `AND`
3. `NOT`（単項）
4. 比較 `=` `<>` `<` `<=` `>` `>=`
5. `+` `-`
6. `*` `/` `MOD`
7. `^`（べき乗、右結合）
8. 単項 `-` `+`
9. 数値 / 文字列 / `( )` / 関数 / 配列参照 / 変数

### 7.3 各演算子の挙動

- `+`: 数値同士は加算。いずれかが文字列なら両辺を文字列化して連結する。
- `-` `*`: 数値演算。
- `/`: 実数除算（常に実数結果）。0 で割るとエラー。
- `MOD`: 整数剰余（両辺を整数化）。0 で割るとエラー。
- `^`: べき乗（右結合）。`2 ^ 3 ^ 2` は `2 ^ (3 ^ 2)`。
- 比較演算: 結果は `-1`（真）/ `0`（偽）。文字列同士の比較も可能。数値と文字列の比較はエラー。
- 論理演算: 値を整数化してビット演算を行う（`AND`=ビット積、`OR`=ビット和、`XOR`=排他的論理和、`NOT`=ビット反転）。真偽値 -1 / 0 と組み合わせると論理演算として機能する。

```basic
10 IF A > 0 AND A < 10 THEN PRINT "1桁"
20 PRINT 7 MOD 3          ' 1
30 PRINT 2 ^ 10           ' 1024
40 PRINT "AB" + "CD"      ' ABCD
```

## 8. 命令（ステートメント）

### LET（代入）
```
[LET] 変数 = 式
[LET] 配列(添字, ...) = 式
```
`LET` は省略可能。

### PRINT
```
PRINT [式] [ ; | , [式] ] ...
```
- `;`（セミコロン）: 区切りなしで連結。
- `,`（カンマ）: タブ位置（タブ幅 8 桁）まで送って次を表示。
- 行末が式で終わる場合は改行する。行末が `;` または `,` の場合は改行しない。

```basic
10 PRINT "X="; X          ' 区切りなし連結し改行
20 PRINT A, B             ' タブ区切り
30 PRINT "WAIT";          ' 改行しない
```

### INPUT
```
INPUT ["プロンプト" ; | ,] 変数 [, 変数 ...]
```
- プロンプト文字列を省略すると `? ` を表示する。
- 複数の変数を指定した場合、入力をカンマで区切って取り込む。
- 数値変数へは数値変換して代入（変換できない場合は 0）。文字列変数へはそのまま代入。

```basic
10 INPUT "NAME"; N$
20 INPUT "X,Y"; X, Y
```

### GOTO
```
GOTO 行番号
```
指定行へ無条件ジャンプ。行番号には式も書ける（整数化される）。

### GOSUB / RETURN
```
GOSUB 行番号
...
RETURN
```
`GOSUB` でサブルーチンを呼び、`RETURN` で呼び出し元の次の行へ戻る。ネスト可能。

### IF ... THEN
```
IF 条件 THEN 行番号
IF 条件 THEN 命令
```
- 条件が真（0 以外）のとき `THEN` 以降を実行する。
- `THEN` の後が行番号のみの場合は、その行へジャンプ（暗黙の GOTO）。
- `THEN` の後が命令の場合は、その命令を実行する。
- `ELSE` は**未対応**。

```basic
10 IF A = 0 THEN 100
20 IF A > 0 THEN PRINT "PLUS"
```

### FOR ... NEXT
```
FOR 変数 = 開始 TO 終了 [STEP 増分]
...
NEXT [変数]
```
- `STEP` 省略時の増分は 1。
- 増分が正なら「変数 > 終了」で、負なら「変数 < 終了」でループを抜ける。
- `NEXT` に変数を指定すると、その変数のループまで巻き戻して処理する（多重ループ対応）。

```basic
10 FOR I = 1 TO 10
20 PRINT I
30 NEXT I

40 FOR Y = 10 TO 0 STEP -2
50 PRINT Y
60 NEXT Y
```

### DIM
```
DIM 配列(サイズ [, サイズ ...]) [, 配列(...) ...]
```
配列を宣言する（[配列](#62-配列)参照）。

### REM
```
REM コメント
```
行末までをコメントとして無視する。

### DATA / READ / RESTORE
```
DATA 定数 [, 定数 ...]
READ 変数 [, 変数 ...]
RESTORE
```
- `DATA` に数値・文字列の定数列を定義する（実行前に全 `DATA` 行が収集される）。
- `READ` は `DATA` から順に値を読み出して変数へ代入する。
- `RESTORE` は読み出し位置を先頭へ戻す。

```basic
10 DATA 10, 20, "HELLO"
20 READ A, B
30 READ C$
40 PRINT A + B; " "; C$    ' 30 HELLO
```

### CLS
```
CLS
```
テキスト画面とグラフィックを消去し、カーソルを左上へ戻す。

### LOCATE
```
LOCATE X, Y
```
カーソル位置を指定する（横 X、縦 Y、ともに 0 始まり）。画面範囲にクランプされる。

### COLOR
```
COLOR 色番号
```
これ以降に出力する文字の色を設定する（[色パレット](#15-付録-色パレット)参照）。

### PSET
```
PSET (X, Y) [, 色番号]
PSET (X, Y, 色番号)
```
点を描画する。色を省略すると現在の `COLOR` 色を使う。

### LINE
```
LINE (X1, Y1)-(X2, Y2) [, 色番号]
```
直線を描画する。色を省略すると現在の `COLOR` 色を使う。

### RANDOMIZE
```
RANDOMIZE [シード]
```
乱数を初期化する。シードを省略すると時刻などで初期化する。

### VSYNC
フレーム制御命令。[フレーム制御](#10-フレーム制御vsync)を参照。

### END / STOP
```
END
STOP
```
プログラムの実行を終了する。

## 9. 組み込み関数

### 9.1 文字列関数

| 関数 | 戻り値 |
|---|---|
| `LEN(s)` | 文字列 s の長さ |
| `LEFT$(s, n)` | s の左端から n 文字 |
| `RIGHT$(s, n)` | s の右端から n 文字（n ≤ 0 なら空文字） |
| `MID$(s, start [, len])` | s の start 文字目（**1 始まり**）から len 文字。len 省略時は末尾まで |
| `CHR$(n)` | 文字コード n に対応する文字 |
| `ASC(s)` | s の先頭文字の文字コード（空文字なら 0） |
| `STR$(n)` | 数値 n を文字列化 |
| `VAL(s)` | 文字列 s を数値化（変換不可なら 0、`.` を含めば実数） |

```basic
10 A$ = "PYXELBASIC"
20 PRINT LEFT$(A$, 5)      ' PYXEL
30 PRINT MID$(A$, 6, 3)    ' BAS
40 PRINT CHR$(65)          ' A
```

### 9.2 数学関数

| 関数 | 戻り値 |
|---|---|
| `ABS(n)` | 絶対値 |
| `SGN(n)` | 符号（n>0 で 1、n=0 で 0、n<0 で -1） |
| `INT(n)` | n 以下の最大の整数（切り捨て、負方向） |
| `FIX(n)` | 小数部を切り捨て（0 方向） |
| `ROUND(n)` | 四捨五入 |
| `SIN(n)` `COS(n)` `TAN(n)` | 三角関数（ラジアン入力） |
| `ATN(n)` | 逆正接（アークタンジェント） |
| `RAD(n)` | 度 → ラジアン |
| `DEG(n)` | ラジアン → 度 |
| `EXP(n)` | e の n 乗 |
| `LOG(n)` | 自然対数 |
| `LOG10(n)` | 常用対数 |
| `SQR(n)` | 平方根 |

> `INT` と `FIX` は負数で挙動が異なる。`INT(-1.5)` は `-2`、`FIX(-1.5)` は `-1`。

### 9.3 乱数

| 関数 | 戻り値 |
|---|---|
| `RND` | 0 以上 1 未満の実数 |
| `RND(n)` | 0 以上 n 未満の実数 |

- 整数乱数は `INT(RND(n))` で 0 ～ n-1 を得る。
- 初期化は `RANDOMIZE` 命令で行う。

### 9.4 入力関数

| 関数 | 戻り値 |
|---|---|
| `INKEY$` | 直近フレームに入力された文字（なければ空文字） |
| `STICK(n)` | 方向キーの状態をビット和で返す（下表） |
| `BUTTON(n)` | ボタン n が押されていれば 1、そうでなければ 0 |

`STICK` の戻り値（複数方向は和になる）:

| 方向 | 値 |
|---|---|
| 上 | 1 |
| 下 | 2 |
| 左 | 4 |
| 右 | 8 |

> 現在の実装では `STICK(n)` の引数 n は無視され、常に矢印キーの状態を返す。

`BUTTON(n)` のキー割り当て:

| n | キー |
|---|---|
| 0 | Z |
| 1 | X |
| 2 | C |
| 3 | スペース |

```basic
10 IF STICK(0) AND 1 THEN Y = Y - 1     ' 上キーで上へ
20 IF BUTTON(0) THEN PRINT "Z PRESSED"
```

## 10. フレーム制御（VSYNC）

PyxelBasic は 60 FPS で動作する。指定した命令・関数に達した時点で「そのフレームの実行を打ち切り、次フレームへ継続する」ことで、画面更新や入力の取りこぼしを防ぐ。

### 10.1 フレーム区切りの仕組み

- フレーム区切り対象の予約語を実行（命令）または評価（関数）すると、そこでフレームが区切られる。
- 初期状態で区切り対象になっている予約語: **`PRINT` `PSET` `LINE` `STICK` `BUTTON`**。
- 区切り対象は `VSYNC` コマンドで実行時に変更できる。

### 10.2 VSYNC コマンド

| 構文 | 動作 |
|---|---|
| `VSYNC` | その場でフレームを区切る（明示的な同期点） |
| `VSYNC 予約語 ON` | 指定した予約語を区切り対象に追加 |
| `VSYNC 予約語 OFF` | 指定した予約語を区切り対象から除外 |
| `VSYNC RESET` | 区切り設定を初期状態に戻す |
| `VSYNC LIST` | 現在の区切り対象を一覧表示する |

- 予約語名が正しくない（未定義の語）場合はエラーになる。
- 設定は実行をまたいで保持され、`VSYNC RESET` または `NEW` で初期状態に戻る。
- 別プログラムの先頭に `VSYNC RESET` を書いておくと、直前の設定変更に影響されず既定状態で動かせる。

```basic
10 VSYNC PRINT OFF       ' PRINT で区切らないようにする（高速一括出力）
20 FOR I = 1 TO 100
30 PRINT I
40 NEXT I
50 VSYNC RESET
```

ゲームのメインループでは、`PSET` や `STICK` が区切り対象なので明示的な `VSYNC` を書かなくても 1 フレーム単位で動く。

## 11. グラフィックとテキスト画面

### 11.1 座標系

- 画面左上が `(0, 0)`。右方向に X が増加、下方向に Y が増加。
- 画面サイズは 256 x 256 ピクセル。

### 11.2 グラフィック

- 描画命令: `PSET`、`LINE`。
- グラフィックは専用のレイヤーに描画され、`CLS` まで保持される（毎フレーム再実行不要）。

### 11.3 テキスト出力と制御文字

- `PRINT` はテキスト画面に文字を出力する。画面下端まで来ると上方向へスクロールする。
- 文字列中に次の制御文字（文字コード）を含めると、出力時に作用する。

| 文字コード | 取得方法 | 作用 |
|---|---|---|
| 改行 (10) | `CHR$(10)` | 改行（次の行の先頭へ） |
| 復帰 (13) | `CHR$(13)` | 行頭へ戻る（改行しない） |
| タブ (9) | `CHR$(9)` | 次のタブ位置（タブ幅 8 桁）へ |

> `\n` `\l` `\cN` などのバックスラッシュ・エスケープは未対応。改行は `CHR$(10)` を使う。

## 12. キーボード入力

| 用途 | 方法 |
|---|---|
| 1 行入力 | `INPUT`（Enter で確定） |
| リアルタイム 1 文字 | `INKEY$`（押されていなければ空文字） |
| 方向キー | `STICK(n)` |
| ボタン | `BUTTON(n)`（Z / X / C / スペース） |

編集モードでは、文字キーで入力し、Backspace で 1 文字削除、Enter で行を確定する。

## 13. エラーメッセージ

実行中のエラーは `?ERROR <コード> in line <行番号>: <内容>`、ダイレクト実行時は `?ERROR <コード>: <内容>` の形式で表示される。メッセージは英語で出力される。各エラーには数値コードがあり、100 番台で分類する（1xx 構文/パース、2xx 型、3xx 実行時/制御フロー、4xx 配列/データ、5xx ファイル/コマンド）。コードとメッセージは `pyxelbasic/errors.py` に一元定義する。

| コード | メッセージ | 主な原因 |
|---|---|---|
| 101 | `Syntax error` | 文として成立しない行（任意のテキスト、`=` の無い暗黙代入など）|
| 102 | `Unsupported statement: x` | 未実装の命令キーワード |
| 103 | `Invalid expression` | 式の構文エラー |
| 104 | `Expected '('` | `(` が必要 |
| 105 | `Expected ')'` | `)` が必要 |
| 106 | `Expected '='` | 代入に `=` がない |
| 107 | `Assignment target is not a variable` | `=` の左辺が変数でない |
| 108 | `Expected THEN` | `IF` に `THEN` がない |
| 109 | `Nothing after THEN` | `THEN` の後ろが空 |
| 110 | `Missing FOR variable` | `FOR` にループ変数がない |
| 111 | `Expected '=' in FOR` | `FOR` の構文エラー |
| 112 | `Expected TO in FOR` | `FOR` の構文エラー |
| 113 | `Invalid DIM syntax` | `DIM` の構文エラー |
| 114 | `Invalid LOCATE syntax` | `LOCATE` の構文エラー |
| 115 | `Invalid LINE syntax ('-' required)` | `LINE` の構文エラー |
| 116 | `VSYNC: keyword required` | `VSYNC` に予約語が必要 |
| 117 | `VSYNC: ON or OFF required` | `VSYNC <語>` に `ON`/`OFF` が必要 |
| 118 | `Unterminated string` | `"` が閉じていない |
| 119 | `Invalid character: 'x'` | 解釈できない文字 |
| 120 | `Invalid comparison operator` | 内部エラー（不正な比較演算子）|
| 201 | `Number required` | 文字列を数値演算に使った |
| 202 | `Type mismatch in comparison` | 数値と文字列を比較した |
| 203 | `Cannot assign string to numeric variable: name` | 代入時の型不一致 |
| 301 | `Division by zero` | 0 で割った（`/` または `MOD`） |
| 302 | `Line n not found` | ジャンプ先の行が存在しない |
| 303 | `RETURN without GOSUB` | `RETURN` が余分 |
| 304 | `NEXT without FOR` | `NEXT` が余分 |
| 305 | `NEXT name without FOR` | `NEXT 変数` に対応する `FOR` がない |
| 401 | `Subscript out of range: name` | 配列の添字が範囲外 |
| 402 | `Out of DATA` | `DATA` を読み切った |
| 501 | `SAVE requires a file name` | `SAVE` にファイル名がない |
| 502 | `LOAD requires a file name` | `LOAD` にファイル名がない |

`?FILE NOT FOUND "名前"` は `LOAD` 先のファイルが無いときに表示される別個の通知で、コード付きエラーではない。

## 14. 未実装・制限事項

現在のプロトタイプでは未対応の項目:

- **制御構造**: `ELSE`（`IF` の偽側）、マルチステートメント（`:`）、`WHILE`/`DO`
- **グラフィック**: `BOX`、`CIRCLE`、`FILLCIRCLE`、`SPRITE`、`SPRITEDEF`、`PCG`、`PALETTE`
- **サウンド**: `PLAY`（MML）、`MUSIC`
- **文字列エスケープ**: `\n` `\l` `\t` `\cN`、`CHR$(12)` によるレトロ風改行、`COLOR` の `\cN` エスケープ指定
- **その他**: PyxelBasic 標準フォントは ASCII の範囲。日本語表示は未対応。
